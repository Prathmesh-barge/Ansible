What is Ansible?

Ansible is an open-source, agentless IT automation tool used for configuration management, deployment, and provisioning.

Simple: Control multiple servers from one machine without installing software on them.

Key Features

FeatureDescription
AgentlessNo software needed on target nodes
Simple SyntaxUses YAML
IdempotentSafe to re-run
Push-BasedSends config from control node
ScalableHandles thousands of servers

Architecture

Control Node → SSH → Managed Nodes
(No agent required)

Core Components

  • Control Node – Where Ansible runs
  • Managed Nodes – Target servers
  • Inventory – List of servers
  • Modules – Task executors
  • Playbooks – YAML automation scripts
  • Roles – Organized structure

How Ansible Works

  1. Create Inventory
  2. Write Playbook
  3. Run command
  4. SSH into servers
  5. Execute tasks

Installation

sudo apt update
sudo apt install ansible -y
ansible --version

Passwordless SSH Setup

ssh-keygen
cat ~/.ssh/id_rsa.pub
# Paste into target authorized_keys

Inventory Example

[webservers]
192.168.1.10
192.168.1.11

Ad-Hoc Commands

ansible -i inventory all -m shell -a "df -h"

Sample Playbook

- name: Install Nginx
  hosts: all
  become: true

  tasks:
    - name: Install nginx
      apt:
        name: nginx
        state: present

Important Modules

ModuleUsage

shellRun commands
